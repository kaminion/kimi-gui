# Kimi-GUI 0.6.1

Version 0.6.1 makes the CLI connection choice more informative and keeps work
adjustments under the user's control until they are delivered. It also
standardizes the product name as **Kimi-GUI** and moves the documentation
toward real, task-based workflows.

## One Kimi-GUI name everywhere

The app name is now consistently shown as **Kimi-GUI** in the startup
wordmark, window title, macOS app metadata, Windows shortcut, installer
filenames, and user documentation. The lowercase `kimi-gui` form remains only
where npm and GitHub require the existing machine-readable identifier.

![Kimi-GUI startup wordmark](https://raw.githubusercontent.com/kaminion/kimi-gui/v0.6.1/docs/media/release-0.6.1-kimi-gui-brand.png)

## See what CLI agent mode adds

The optional connection dialog now explains the features available after
connecting Kimi Code CLI:

- Swarm and sub-agents for parallel work
- Plan mode for reviewing the approach before edits
- The full Kimi Code CLI toolset
- Continuity with sessions created by Kimi Code CLI

The same dialog still detects whether to show **Connect CLI** or **Install
CLI**, and keeps the **Do not show again** option.

![CLI agent mode capability guide](https://raw.githubusercontent.com/kaminion/kimi-gui/v0.6.1/docs/media/release-0.6.1-cli-capabilities.png)

## Edit or delete queued work adjustments

Steering messages now enter a short queued state before the engine consumes
them. Each queued row exposes **Edit** and **Delete** actions. Opening the
editor pauses delivery, Save restarts the delivery window with the revised
text, and Cancel returns the original message to the queue.

The behavior is supported by both engines. The built-in engine holds the
adjustment until a safe model boundary, while CLI agent mode uses the daemon's
queued prompt and steering APIs.

![Queued work adjustment with Edit and Delete actions](https://raw.githubusercontent.com/kaminion/kimi-gui/v0.6.1/docs/media/release-0.6.1-queued-steer.png)

## Additional details

- Queued adjustments have a four-second grace period.
- Delivery controls lock as soon as steering begins.
- A failed CLI promotion remains visible and editable instead of disappearing.
- Leaving an open editor safely resumes the held adjustment.
- Dark and light themes share the same interaction states and focus treatment.
- README and the usage guide now lead with concrete project, prompting,
  steering, and change-review workflows.

## Downloads

- macOS Apple Silicon: DMG or ZIP
- macOS Intel: DMG or ZIP
- Windows x64: NSIS installer or portable executable

Development builds are unsigned. macOS may show a Gatekeeper warning, and
automatic installation requires a properly signed production build.
