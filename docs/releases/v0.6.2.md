# Kimi-GUI 0.6.2

Version 0.6.2 makes the Kimi-GUI product name consistent across every Windows
surface and adds regression coverage for the runtime and packaging metadata.
It also includes the maintainability improvements completed after 0.6.1.

## Kimi-GUI everywhere on Windows

Windows packages now explicitly use **Kimi-GUI** instead of falling back to
the npm-safe package identifier:

- The installed and portable process is `Kimi-GUI.exe`.
- File description, internal name, and product name are `Kimi-GUI`.
- Start menu and desktop shortcuts use `Kimi-GUI`.
- Apps & Features shows `Kimi-GUI` as the uninstall entry.
- Taskbar grouping and notifications use the stable `com.kimi.gui`
  AppUserModelID.
- The first BrowserWindow title is set before the renderer finishes loading.

These values are now covered by an automated branding contract test so a
future packaging change cannot silently reintroduce a lowercase or shortened
name.

![Kimi-GUI startup wordmark](https://raw.githubusercontent.com/kaminion/Kimi-GUI/v0.6.2/docs/media/release-0.6.1-kimi-gui-brand.png)

## Maintenance and reliability

- Extracted built-in-engine steering timing and cancellation into a focused
  queue module.
- Added regression tests for queued steering in both the built-in and CLI
  engines.
- Added a cross-platform `npm run check` command for all shipped JavaScript
  plus the test suite.
- Corrected the English change summary from `1 files changed` to
  `1 file changed`.
- Updated the architecture guide to describe current module boundaries,
  validation, and branding ownership.

## Downloads

- macOS Apple Silicon: DMG or ZIP
- macOS Intel: DMG or ZIP
- Windows x64: NSIS installer or portable executable

Development builds are unsigned. macOS may show a Gatekeeper warning, and
automatic installation requires a properly signed production build.
